## 指令优先级

设计技能覆盖默认系统提示行为，但**用户指令始终具有最高优先级**：

1. **用户的明确指令**（CLAUDE.md、GEMINI.md、AGENTS.md、直接请求）——最高优先级
2. **设计技能** ——在冲突处覆盖默认系统行为
3. **默认系统提示** ——最低优先级

## 规则

**在任何响应或操作之前调用相关或被请求的技能。** 哪怕只有 1% 的可能性某个技能适用，你都应该调用该技能来检查。如果调用后发现技能不适合当前情况，你不需要使用它。

## 红线

这些想法意味着停下——你在合理化：

| 想法 | 现实 |
|------|------|
| "这只是一个简单的问题" | 问题就是任务。检查技能。 |
| "我需要先了解更多上下文" | 技能检查在澄清性问题之前。 |
| "让我先探索一下代码库" | 技能告诉你如何探索。先检查。 |
| "这不需要正式的技能" | 如果技能存在，就使用它。 |
| "我记得这个技能" | 技能会迭代更新。阅读当前版本。 |
| "技能太小题大做了" | 简单的事会变复杂。使用它。 |
| "让我先做这一件事" | 在做任何事之前先检查。 |

## 技能优先级

当多个技能可能适用时，使用此顺序：

1. **流程技能优先**（头脑风暴、调试）- 这些决定如何处理任务
2. **设计技能其次**（design-taste-frontend、风格预设）- 这些指导执行

"帮我设计一个落地页" -> 先头脑风暴，再使用 design-taste-frontend。
"把这个现有项目改好看" -> 先使用 redesign-existing-projects，再配合风格预设。

## 风格路由

当用户没有指定风格方向时，根据项目类型自动选择：

| 项目类型 | 推荐风格预设 |
|----------|-------------|
| SaaS / AI 工具 / 开发者平台 | design-taste-frontend（默认）或 high-end-visual-design |
| 生活方式 / 编辑 / 杂志 | minimalist-ui |
| 数据仪表盘 / 安全 / 军工风 | industrial-brutalist-ui |
| 品牌视觉 / Logo 系统 | brandkit |
| Google Stitch 屏幕 | stitch-design-taste |
| 现有项目改造 | redesign-existing-projects |

用户明确指定风格时，以用户选择为准。

## 技能类型

**刚性的**（品牌规范、可访问性）：严格遵循。不要偏离纪律。

**灵活的**（风格预设、布局模式）：根据上下文调整原则。

技能本身会告诉你它属于哪种。

## 用户指令

指令说明做什么，而非怎么做。"添加 X"或"修复 Y"不意味着跳过工作流。

# 设计技能索引

## design-taste-frontend

**触发：** 构建 landing page、portfolio、营销站点或任何前端界面时使用——反 AI-slop 设计系统，三旋钮架构（方差/动效/密度），自动推断设计方向，审计前优先。

**适用：** 新建前端页面；需要排版、颜色、布局、暗模式的完整设计系统；需要防止 AI 生成廉价/模板化的界面。

**不适用：** 纯 Flutter/iOS 原生开发（用 using-flutter/using-dev）；纯后端或基础设施。

## high-end-visual-design

**触发：** 需要 $150k 级代理品质的 UI——双面嵌套架构、磁力微物理、极致动效编排。

**适用：** 高端 SaaS 落地页；Awwwards 级作品集；需要"看起来很贵"的界面。

**不适用：** 低调/极简/功能性工具界面；信息密集的数据仪表盘。

## minimalist-ui

**触发：** 需要温暖单色、编辑风、极简的 UI——扁平排版、柔和色调、安静精致。

**适用：** 编辑型站点；温暖极简的产品页；需要"安静但有品质感"的界面。

**不适用：** 需要强视觉冲击的营销页；数据密集的仪表盘；暗色系科技风。

## industrial-brutalist-ui

**触发：** 需要原始机械感界面——瑞士排版 + CRT 终端美学，刚性网格，极端字重对比，模拟退化效果。

**适用：** 数据密集仪表盘；军工/航天主题 UI；需要"解密蓝图"感的作品集或编辑站点。

**不适用：** 温暖/友好/消费级产品；需要圆润和柔和的界面。

## redesign-existing-projects

**触发：** 升级现有网站或应用到高品质——审计当前设计，识别 AI 通用模式，逐项修复而不破坏功能。

**适用：** 现有项目需要视觉升级；AI 生成的页面需要"去模板化"；与任何 CSS 框架或原生 CSS 配合。

**不适用：** 全新项目从零开始（用 design-taste-frontend 或风格预设）；纯后端重构。

## brandkit

**触发：** 生成品牌视觉系统图片——Logo、品牌指南板、视觉世界展示，高端品牌识别系统。

**适用：** 需要生成品牌 Kit 图片；Logo 概念和品牌视觉世界；品牌系统跨 UI/印刷/细节的展示。

**不适用：** 写前端代码（用 design-taste-frontend）；不需要图片生成的场景。

## stitch-design-taste

**触发：** 为 Google Stitch 生成 DESIGN.md 语义设计系统文件——将反 AI-slop 设计指令翻译为 Stitch 原生的语义设计语言。

**适用：** 使用 Google Stitch 生成屏幕；需要为 Stitch 准备设计系统文件；Stitch MCP 集成。

**不适用：** 不使用 Google Stitch 的常规前端开发（用 design-taste-frontend）。
