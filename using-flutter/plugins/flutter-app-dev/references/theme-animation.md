# 主题与动画

## 主题统一原则

- 全局视觉统一放在 `ThemeData`。
- 局部差异用 `Theme.of(context).copyWith(...)` 或组件级主题覆盖。
- 不要把颜色、圆角、字号散落在每个页面里手写。

## Material 3 建议

- 新项目默认按 Material 3 思路组织主题。
- 优先从 `ColorScheme` 出发，而不是手工散落维护一堆颜色。
- 组件风格优先通过组件主题统一，不在单个按钮上重复写样式。

## 推荐做法

### 颜色

- 先定义品牌主色和语义色。
- 语义色优先表示状态：成功、警告、错误、信息。
- 文本和背景始终考虑对比度。

### 字体

- 使用统一的 `TextTheme`。
- 不同层级文本只做必要变体，不在页面里随手造样式。

### 组件

- Button、Card、Input、Dialog 等公共组件尽量主题化。
- 对于复用频繁的样式，抽成主题或公共 Widget。

## 动画选型

### 隐式动画

适合：

- 尺寸、颜色、透明度、圆角变化。
- 状态切换带来的轻量动效。

常用：

- `AnimatedContainer`
- `AnimatedOpacity`
- `AnimatedSwitcher`
- `TweenAnimationBuilder`

### 显式动画

适合：

- 需要暂停、反向、串联、分段控制。
- 需要精细控制时间轴。

常用：

- `AnimationController`
- `AnimatedBuilder`
- `CurvedAnimation`

### 转场和共享元素

适合：

- 页面间元素过渡。
- 自定义页面切换动效。

常用：

- `Hero`
- `PageRouteBuilder`

## 选择规则

- 能用隐式动画解决，就不要先上显式动画。
- 动画服务于反馈，不服务于炫技。
- 如果动画影响滚动、点击或阅读，就先减弱。

## 稳定性规则

- 显式动画必须在 `dispose()` 中清理控制器。
- 不要在一个复杂页面里同时叠加多种高频动画。
- 列表、表单和信息密集页面优先稳定与可读。

## 交互反馈优先级

1. 用户知道操作是否生效。
2. 用户知道页面是否在切换或加载。
3. 用户不会因为动画丢失上下文。

## 常见反模式

- 每个按钮单独写一套颜色和圆角。
- 主题没统一，靠复制粘贴样式“保持一致”。
- 明明只是状态切换，却直接使用多控制器显式动画。
- 动画时长过长，阻碍连续操作。
- 页面性能还没稳定，就先上复杂转场。

## 快速检查

- [ ] 样式是否主要由主题控制。
- [ ] 组件状态色是否统一。
- [ ] 动画方案是否匹配需求复杂度。
- [ ] 显式动画控制器是否正确释放。
- [ ] 动画是否影响阅读、滚动和点击稳定性。
