# ArkTS 专项批次2练习清单

## 主题

1. 状态分层（页面状态、共享状态、持久化状态）
2. Want 冷热启动统一处理
3. 并发任务（TaskPool 思路）与主线程减负
4. 混编参数透传与生命周期一致性

## A 组：状态分层

### A1. 状态盘点

- 目标：从一个样例中列出全部状态变量，并分层归类。
- 建议样例：`ohos/flutter_ohos_theme_fontsizescale`、`ohos/ohos_themeAdaptation`
- 验收：
  - 输出“状态变量 -> 所属层级 -> 生命周期 -> 存储方式”表格。

### A2. 局部状态去全局化

- 目标：挑一个页面，把不必要的共享状态改为页面局部状态方案（设计层面）。
- 验收：
  - 说明为什么该状态不应该放共享层。

## B 组：Want 冷热启动统一

### B1. 冷启动参数路径

- 目标：梳理从启动参数到页面消费的链路。
- 建议样例：`ohos/test_uni_links/packages/x_uni_links`
- 验收：
  - 标注关键函数入口与数据字段。

### B2. 热启动参数路径

- 目标：梳理 `onNewWant` 路径，给出与冷启动统一策略。
- 验收：
  - 明确“首次参数”“后续参数”冲突时的处理规则。

### B3. Flutter 透传设计

- 目标：设计 ArkTS -> Flutter 的参数透传结构（键名、类型、默认值）。
- 验收：
  - 给出字段契约与兼容策略。

## C 组：并发与性能

### C1. 可并发任务识别

- 目标：从样例中识别 3 类可并发任务（重计算、批量 IO、解析）。
- 建议入口：`ohos/docs/05_performance/`
- 验收：
  - 每类任务给出“放主线程的风险”。

### C2. 任务边界设计

- 目标：为一个重任务设计输入/输出数据结构，避免直接操作 UI 对象。
- 验收：
  - 输出“任务函数签名草案 + 回调/结果处理点”。

### C3. 混编性能关注点

- 目标：说明并发任务与 Flutter 生命周期交互时的注意点。
- 验收：
  - 给出“页面销毁后结果回调”的兜底方案。

## D 组：混编一致性

### D1. 生命周期一致性核对

- 目标：核对 `aboutToAppear/aboutToDisappear/onPageShow/onPageHide` 与 Flutter 引擎状态操作的对应关系。
- 建议样例：`add_to_app/multiple_flutters/multiple_flutters_ohos`
- 验收：
  - 输出一张“ArkTS 生命周期 -> 引擎动作”映射表。

### D2. 通道与参数契约

- 目标：梳理一个样例的 Channel 参数类型并补充错误分支策略（设计层面）。
- 建议样例：`ohos/channel_demo`
- 验收：
  - 给出参数校验与错误码约定。
