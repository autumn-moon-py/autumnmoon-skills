# 测试

## 默认策略

- 新单元测试优先使用 Swift Testing。
- UI 测试、系统交互、性能测试继续使用 XCTest / XCUITest。

## Swift Testing 适用范围

适合：

- 纯逻辑测试。
- 服务层测试。
- 并发逻辑测试。
- 参数化测试。

常用要点：

- `@Test`
- `#expect`
- `#require`
- 参数化输入

## XCTest / XCUITest 保留场景

适合：

- UI 自动化。
- 启动流程。
- 原生权限弹窗与系统交互。
- 性能基准。

## 建议覆盖面

### 逻辑层

- 状态变化。
- 错误路径。
- 取消行为。
- 重试行为。

### 网络层

- 成功响应。
- 非 2xx 响应。
- 解码失败。
- Token 失效与刷新。

### 视图层

- 关键状态渲染。
- 交互是否触发正确动作。
- 复杂导航与 Sheet 是否按预期工作。

## 异步测试重点

- 不依赖拍脑袋的等待时间。
- 验证取消而不只验证成功。
- 明确主线程回写是否发生在正确时机。

## 常见错误

- 逻辑层全靠 UI 测试间接覆盖。
- 异步测试只测成功路径。
- 用漫长 `sleep` 支撑脆弱测试。

## 快速检查

- [ ] 单元测试优先使用 Swift Testing。
- [ ] UI / 系统流程保留 XCTest。
- [ ] 并发、取消、错误路径有专门验证。
