# 导航与页面组合

## `NavigationStack` 优先

新页面默认优先使用 `NavigationStack`。

适合：

- 推栈式页面流转。
- 类型清晰的路由。
- 可演进的原生导航结构。

建议：

- 用明确的路由类型表示页面目的地。
- 程序化跳转和用户点击跳转共用同一套路由语义。

## Sheet 规则

- 如果 Sheet 表示“当前选中的实体”，优先使用 `.sheet(item:)`。
- Sheet 自己负责保存、取消、关闭，不把所有动作抛回父视图拼接。
- 有未保存改动时，显式处理关闭确认。

## Tab 规则

- 每个 Tab 自己维护导航栈。
- Tab 负责顶层结构，页面内不要再手搓另一套隐式路由系统。
- 特殊 Tab（如发布按钮）可以通过选择拦截触发动作，但逻辑要显式。

## 深链

深链需要统一入口：

- URL 解析。
- 路由映射。
- 未登录、无权限、目标不存在时的回退逻辑。

规则：

- 深链解析不要散落到多个页面。
- 路由失败时给出明确降级路径。

## 页面组合

- 先拆主结构，再拆局部组件。
- 把重复出现的头部、卡片、操作区抽成小组件。
- 让组合服务于可读性，而不是机械拆文件。

## 不要过度设计

以下情况通常不需要专门 Router 对象：

- 只有 2 到 3 层简单跳转。
- 没有复杂回退或深链。
- 没有多入口跳到同一目标的要求。

## 常见错误

- 页面不复杂，却过早引入全局 Router。
- 一个 Tab 内混用多套导航状态。
- Sheet 的保存与关闭逻辑分散在父子多处。
- 深链入口和页面内部跳转使用不同语义。

## 快速检查

- [ ] 当前导航复杂度是否真的需要 Router。
- [ ] Tab 是否各自拥有独立导航栈。
- [ ] Sheet 关闭与未保存处理是否清楚。
- [ ] 深链解析是否只有一个统一入口。
